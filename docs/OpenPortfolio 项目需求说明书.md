# OpenPortfolio 项目需求说明书

本文档基于 OpenPortfolio 项目定位（基于 React + TypeScript + Ant Design 的通用开发者作品集项目，支持配置化快速生成个人工程作品展示网站），明确项目核心目标、功能需求、非功能需求及约束条件，为项目开发提供清晰依据。

# 一、项目核心目标

本项目核心目标是打造一款「配置化驱动、低门槛使用、高通用性」的开发者作品集生成工具，具体实现以下核心价值：

1. 低门槛上手：开发者无需修改核心代码，仅通过配置文件即可自定义网站内容与样式，快速生成个性化作品集网站；

2. 高通用性适配：兼容前端、后端、全栈、移动端等不同领域开发者的展示需求，支持灵活扩展；

3. 技术栈落地：基于 React + TypeScript 保障组件化开发与类型安全，依托 Ant Design 实现美观、统一且响应式的 UI 体验；

4. 轻量化部署：支持静态打包输出，可直接部署于 Github Pages、Netlify、Vercel 等平台，无需依赖后端服务。

# 二、核心功能需求

功能需求以「配置化」为核心，覆盖「基础配置-内容展示-交互体验-扩展能力」四大维度，确保满足开发者从搭建到使用的全流程需求。

## （一）基础配置功能

提供全面且易用的配置项，支持开发者快速自定义网站全局属性，配置文件采用 JSON/TS 格式，附带清晰注释与示例，降低修改难度。具体包括：

1. 全局站点配置：支持自定义网站标题、副标题、浏览器图标（favicon）、默认主题（亮色/暗色）、全局配色方案（主色调、辅助色、中性色）、页脚基础信息（版权文本、备案号）；

2. 个人信息配置：支持配置头像（本地/在线链接）、姓名、职业标签（如「前端工程师」「全栈开发者」）、一句话简介、详细个人描述、联系方式（邮箱、Github、Gitee、掘金、知乎等开发者常用平台链接，支持按需显示/隐藏）；

3. 导航配置：支持自定义导航栏菜单名称、对应路由路径、显示顺序，支持配置某一菜单是否隐藏，适配桌面端/移动端布局；

4. 样式细节配置：支持自定义字体大小、卡片圆角/阴影、间距规格、按钮样式等，基于 Ant Design 主题定制能力实现全局样式统一。

## （二）内容展示功能

聚焦开发者核心展示需求，提供结构化的内容展示模块，所有内容均通过配置文件注入，支持灵活增减与修改。具体包括：

1. 作品核心展示模块（核心功能）：

    - 支持配置作品基础信息：名称、分类（如「前端项目」「后端项目」「开源贡献」「实战项目」，分类可自定义）、完成时间、核心描述（简洁说明项目价值与功能）；

    - 支持配置技术栈标签：如「React + TypeScript + AntD」「Vue + Node.js + MongoDB」等，标签颜色可自定义；

    - 支持配置作品关联资源：项目截图（单张/多张，支持懒加载）、在线演示地址、源码仓库地址；

    - 支持配置项目亮点：核心功能模块、技术难点解决方案、个人贡献、项目成果（如「开源项目 100+ Star」「上线服务 1000+ 用户」）。

2. 个人履历展示模块：
        

    - 教育经历：支持配置学校名称、专业、入学/毕业时间、学历、备注（如「优秀毕业生」「校级奖学金」）；

    - 工作经历：支持配置公司名称、职位、入职/离职时间、工作描述（核心职责、项目成果、技能应用）；

    - 技能栈展示：支持按分类（如「前端技术」「后端技术」「数据库」「工具/其他」）展示技能，配置技能名称与掌握程度（如「精通」「熟练」「了解」），支持进度条/标签云等展示形式。

3. 补充内容展示模块（可选配置）：
       

    - 博客/文章关联：支持配置外部博客链接（如掘金、知乎专栏）或内置简单文章列表（标题、摘要、发布时间、链接）；

    - 额外成就展示：支持配置获奖情况、技能证书、开源贡献统计（如「Github 贡献量」「参与开源项目列表」）；

    - 个人寄语：支持配置技术理念、未来规划等个性化文本内容。

## （三）交互与体验功能

优化用户访问体验与开发者使用体验，提供流畅、适配多端的交互效果：

1. 响应式适配：自动适配桌面端、平板、手机端布局，导航栏在移动端自动转为折叠菜单（抽屉组件），作品列表自动调整列数（桌面端 3-4 列、平板 2 列、手机 1 列）；

2. 主题切换：支持亮色/暗色主题手动切换，可选「跟随系统主题」自动适配，切换后全局组件样式同步更新；

3. 内容筛选：作品列表支持按自定义分类筛选（无刷新切换），提升访问者查找效率；

4. 平滑滚动：页面内锚点导航（如从首页跳转至作品区、履历区）实现平滑过渡，提升浏览流畅度；

5. 加载优化：实现图片懒加载、静态资源打包分割，减少首次加载体积，提升页面加载速度；

6. 空状态处理：当某模块无配置内容时（如无作品、无履历），显示友好提示文案（可自定义），避免页面空白。

## （四）扩展性需求

支持开发者根据个性化需求扩展功能，降低二次开发成本：

1. 组件可插拔：核心模块（作品卡片、履历时间线、技能进度条）支持通过配置项启用/隐藏；

2. 自定义组件入口：预留组件扩展接口，支持开发者引入自定义组件（如个人特色展示模块）并通过配置注入页面；

3. 多语言支持（可选）：支持配置中文/英文双语内容，实现语言切换功能，满足海外开发者需求；

4. 配置校验：提供配置文件校验功能，当配置格式错误或缺失必填项时，给出明确错误提示，帮助开发者快速排查问题。

# 三、非功能需求

保障项目的稳定性、易用性、性能与可维护性，满足生产环境使用要求：

## （一）性能需求

1. 加载性能：在常规网络环境下，首页首次加载时间 < 3s，后续页面切换无明显卡顿；

2. 打包优化：静态打包后体积可控，支持 Gzip/Brotli 压缩，核心依赖按需加载；

3. 渲染性能：列表滚动流畅，组件切换无闪烁，图片加载不阻塞页面渲染。

## （二）易用性需求

1. 配置友好：配置文件结构清晰，附带详细注释与完整示例，开发者可直接复制修改，无需理解核心代码逻辑；

2. 文档完善：提供快速上手文档，包含环境搭建、配置说明、部署步骤、常见问题排查等内容；

3. 错误提示：配置错误、打包失败时，给出清晰、可操作的错误提示，降低排查难度。

## （三）可维护性需求

1. 代码规范：基于 TypeScript 实现全量类型定义，组件化拆分合理，代码注释完整，遵循 React 最佳实践；

2. 目录结构清晰：核心代码、配置文件、静态资源、文档等按功能分类存放，便于后续迭代与维护；

3. 依赖稳定：选用官方维护的成熟依赖，避免使用过时或无人维护的包，降低版本兼容风险。

## （四）兼容性需求

1. 浏览器兼容：兼容 Chrome、Firefox、Safari、Edge 最新 3 个版本，兼容主流国产浏览器（如 360、QQ 浏览器）最新版本；

2. 设备兼容：适配主流桌面端分辨率（1366×768 及以上）、平板（iPad、安卓平板）、手机（iOS/Android 主流机型，分辨率 320px 及以上）；

3. 部署兼容：支持部署于各类静态资源服务平台，无需依赖特定后端环境。

# 四、项目约束条件

1. 技术栈约束：必须基于 React + TypeScript + Ant Design 开发，不得替换核心技术栈；

2. 部署约束：最终产物为静态网站，不依赖后端服务，支持无后端环境部署；

3. 配置方式约束：核心配置必须通过 JSON/TS 文件实现，不得要求开发者修改核心业务代码；

4. UI 风格约束：基于 Ant Design 设计体系，保持 UI 风格统一、简洁、专业，符合开发者作品集的定位。

# 五、核心页面规划与功能说明

基于项目核心需求，规划 5 个核心页面（含移动端自适应布局）及全局公共组件，页面功能与配置化需求深度绑定，确保开发者可通过配置快速填充内容，同时保障访问者浏览体验。

## （一）首页（Hero 页面）

1. 页面定位：网站入口，快速传递开发者核心信息，吸引访问者继续浏览，建立初步认知；

2. 核心内容（均为配置化）：个人头像（支持圆形裁剪、尺寸配置）、姓名、职业标签、一句话个人简介、联系方式快捷图标（邮箱、Github 等，支持按需显示/隐藏）；

3. 核心功能：导航快捷跳转（快速链接至作品区、履历区，支持平滑滚动）、主题切换按钮（悬浮/固定展示，支持亮色/暗色切换）；

4. 适配要求：移动端全屏展示核心信息，按钮布局适配小屏操作。

## （二）作品展示页（核心页面）

1. 页面定位：展示开发者工程作品，是作品集的核心价值页面，突出技术能力与项目经验；

2. 核心内容（均为配置化）：作品分类筛选栏（按自定义分类展示，如「前端项目」「开源项目」）、作品卡片列表（每张卡片含作品封面图、名称、分类标签、核心描述、技术栈标签）；

3. 核心功能：分类筛选（无刷新切换作品列表）、作品卡片交互（悬浮阴影加深+轻微上浮效果）、作品关联链接跳转（查看演示、查看源码）、空状态友好提示（无作品配置时显示自定义文案）；

4. 适配要求：响应式网格布局（桌面端 3-4 列、平板 2 列、手机 1 列）、图片懒加载。

## （三）个人履历页

1. 页面定位：展示开发者教育、工作背景及技能栈，辅助印证个人能力，增强可信度；

2. 核心内容（均为配置化）：标签页切换（「履历信息」「技能栈」两大标签，支持配置隐藏某一标签）、教育/工作经历时间线（含学校/公司名称、时间、职位/学历、描述等）、技能栈展示（按分类陈列，含技能名称、掌握程度）；

3. 核心功能：标签页无刷新切换、技能掌握程度可视化（进度条/标签云形式，配置化选择）；

4. 适配要求：时间线布局在移动端适配屏幕宽度，技能展示区分栏自适应（小屏单列展示）。

## （四）作品详情页（扩展页面，可选配置）

1. 页面定位：针对重要作品展示详细信息，深化项目价值与技术深度，面向对作品感兴趣的访问者（如面试官、合作方）；

2. 核心内容（均为配置化）：作品头部信息（名称、分类、完成时间、技术栈标签）、作品封面图（大图+轮播，多张截图时支持切换）、详细描述（项目背景、核心功能、技术实现、难点解决方案、项目成果）；

3. 核心功能：锚点导航（快速跳转至详细描述各段落）、固定操作按钮（返回作品列表、查看演示、查看源码）；

4. 适配要求：长文本分段清晰，图片轮播适配移动端滑动操作。

## （五）关于我页（补充页面）

1. 页面定位：展示更全面的个人信息，拉近距离，增强访问者对开发者的立体认知；

2. 核心内容（均为配置化）：详细个人简介（比首页更完整的自我描述，如发展历程、技术理念）、补充信息（完整联系方式、获奖情况、证书、博客地址等）、个人寄语（技术感悟、未来规划等个性化内容）；

3. 核心功能：联系方式链接直接跳转（邮箱发送、社交平台访问）；

4. 适配要求：文本为主、图片为辅，布局简洁温馨，适配移动端阅读。

## （六）全局公共组件（跨页面复用）

1. 导航栏：桌面端顶部固定（含网站标题、导航菜单），移动端折叠为抽屉菜单（点击展开），支持路由跳转与当前页面高亮；

2. 页脚：全局底部固定，展示配置化版权信息、备案号、联系方式快捷图标，样式统一；

3. 主题切换组件：全局复用，支持亮色/暗色手动切换及跟随系统主题，切换后所有页面组件同步适配。
